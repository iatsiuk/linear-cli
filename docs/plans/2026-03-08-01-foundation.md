# Foundation: Project Infrastructure

## Overview
- Set up core project infrastructure: config management, GraphQL HTTP client, authentication, cobra CLI skeleton, output formatting
- This is the base layer that all subsequent plans depend on
- No Linear-specific business logic - only reusable infrastructure

## Context
- Greenfield Go 1.25 project with minimal `main.go`
- Build system: Makefile with lint/test/build targets
- Auth: Personal API key stored in `~/.config/linear-cli/config.yaml`, override via `LINEAR_API_KEY` env
- CLI framework: cobra with resource-oriented commands (`linear <resource> <action>`)
- Output: table (default) + JSON (`--json` flag)
- API endpoint: POST `https://api.linear.app/graphql`

## Technical Details

### Go Scalar Mapping (from linear-api.md)
| GraphQL | Go | Notes |
|---------|-----|-------|
| `ID` / `String` | `string` | |
| `Float` (nullable) | `*float64` | priority/estimate are ints semantically |
| `Float!` (non-null) | `float64` | |
| `Int` (nullable) | `*int` | |
| `Int!` (non-null) | `int` | |
| `Boolean` (nullable) | `*bool` | |
| `Boolean!` (non-null) | `bool` | |
| `DateTime` | `string` or `time.Time` | ISO 8601, Go: `time.RFC3339` |
| `TimelessDate` | `string` | `YYYY-MM-DD`, Go: `time.DateOnly` |
| `JSON` / `JSONObject` | `json.RawMessage` | |
| `UUID` | `string` | |

### Nullability/Tri-state for Update Inputs
Use `map[string]any` for mutation variables to handle tri-state:
- **omit key** = don't change field
- **key: nil** = clear/unset field (sends `null`)
- **key: value** = set field

### Mutation Payload Pattern
All mutations return payload with `success: Boolean!`. Always check `success`
before accessing the entity field. Entity field names vary by type
(`issue`, `project`, `comment`, etc.). Archive payloads use `entity` field name.

### Rate Limiting
- 5,000 requests/hour per API key
- 250,000 complexity points/hour
- Key header: `X-RateLimit-Requests-Reset` (UTC epoch ms)
- Rate limit errors: HTTP 400, `extensions.code: "RATELIMITED"`
- Backoff: exponential starting at 1s; 5xx: retry with jitter 1-5s
