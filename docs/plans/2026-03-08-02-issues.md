# Issues: Core Issue Management

## Overview
- Implement full CRUD for Linear issues - the primary entity in Linear
- Commands: `linear issue list`, `linear issue show`, `linear issue create`, `linear issue update`, `linear issue delete`
- Depends on Foundation plan (config, API client, output formatter, cobra root)

## Context
- Issue is the core entity with ~50+ fields
- Key fields for CLI: id, identifier, title, description, state, priority, assignee, team, labels, dueDate, estimate, url
- IssueCreateInput requires `teamId` (only required field), optional: title, description, assigneeId, stateId, priority, labelIds, dueDate, estimate, cycleId, projectId, parentId
- IssueUpdateInput: all fields optional (title, description, assigneeId, stateId, priority, labelIds, dueDate, estimate, etc.)
- Filtering via IssueFilter with comparators for all fields
- Pagination: relay cursor-based (first, after, PageInfo)
- **Type Gotchas**: `Issue.priority` is `Float!` on query but `Int` on input (always 0-4). `Issue.estimate` is `Float` on query but `Int` on input. `Issue.number` is `Float!` but always integer.
- **Trashed vs Archived**: `issueDelete` = soft-delete (trash, 30-day grace). `issueArchive` = archive (hidden from default queries). `issueArchive(trash: true)` = same as delete. `issueUnarchive` = restore.
- **Nullability**: use `map[string]any` for update inputs. Omitting key = no change, `nil` = clear value, value = set.
- **Mutation payloads**: always check `success` before accessing entity. `issueCreate`/`issueUpdate` return `IssuePayload` with nullable `issue` field.

## Technical Details

### Key GraphQL fragments
```graphql
fragment IssueFields on Issue {
  id identifier title description
  priority priorityLabel estimate dueDate url
  createdAt updatedAt
  state { id name color type }
  assignee { id displayName email }
  team { id name }
  labels { nodes { id name color } }
}
```

### Issue list query
```graphql
query IssueList($first: Int, $after: String, $filter: IssueFilter) {
  issues(first: $first, after: $after, filter: $filter) {
    nodes { ...IssueFields }
    pageInfo { hasNextPage endCursor }
  }
}
```

### Table columns for `issue list`
| Column | Field | Width |
|--------|-------|-------|
| ID | identifier | 10 |
| Title | title | 40 (truncated) |
| Status | state.name | 15 |
| Priority | priorityLabel | 10 |
| Assignee | assignee.displayName | 20 |
